# Operating Model

This system converts messy initiation material into an authorization-ready project charter through five controlled stages.

## Stage 1: Artifact intake

The agent accepts early ideas, approved business cases, notes, spreadsheets, project plans, CSVs, Word documents, stakeholder inputs, and other source artifacts. It creates an evidence inventory before drafting.

## Stage 2: Initiation diagnosis

The agent evaluates whether the project is ready for chartering. A project is ready for chartering when there is a defined business problem or opportunity, an accountable sponsor, a preferred solution concept, and a need for non-trivial cross-functional execution.

If the idea is still exploratory, the agent should recommend an intake brief or options analysis rather than pretending a charter is ready.

## Stage 3: Adaptive interview

The agent asks targeted questions based on gaps, not a fixed survey. The interview is designed to locate missing facts and force decisions where the draft would otherwise be vague.

## Stage 4: Charter generation

The agent produces a charter that establishes authorization, scope, ownership, execution guardrails, and governance discipline. It avoids rearguing the business case.

## Stage 5: Quality review and packaging

The agent reviews the charter against the readiness rubric, identifies defects, proposes fixes, and produces polished Markdown, HTML, and DOCX outputs.
