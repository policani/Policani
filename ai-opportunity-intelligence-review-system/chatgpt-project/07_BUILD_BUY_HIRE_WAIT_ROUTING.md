# Build Buy Hire Wait Routing

Use these route definitions:

- **Prototype:** The value hypothesis is plausible and a small proof can test it cheaply.
- **Build:** The workflow is differentiated, strategically important, and not well served by available tools.
- **Buy:** The workflow is common, vendor solutions exist, and differentiation comes from adoption rather than custom build.
- **Automate with existing tools:** Current tools can solve enough of the problem with configuration, scripts, or workflow packaging.
- **Hire or upskill:** The gap is human capability, ownership, evaluation design, workflow engineering, or change leadership.
- **Wait:** Market, model, data, or control maturity is too unstable for committed investment.
- **Process-first:** The workflow is too unclear or broken for AI to help safely.
- **Decompose further:** The idea bundles multiple workflows or lacks enough clarity to route.
- **Stop:** The problem, value, owner, or evidence is too weak to justify more attention.

Always include route rationale and the next smallest useful action.
