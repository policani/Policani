# Experiment Design

For promising opportunities, define a lightweight proof plan.

Include:

- Hypothesis
- Test users
- Test data
- Baseline
- Success measures
- Failure measures
- Timebox
- Human review role
- Minimum output
- Stop condition
- Scale condition
- Decision owner
- Intelligence-stack component being tested

Prefer the cheapest credible proof. A prototype is not always the cheapest proof.
