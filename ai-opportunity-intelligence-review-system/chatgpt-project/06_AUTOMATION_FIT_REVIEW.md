# Automation Fit Review

AI is not always the right answer.

Check whether the problem is better handled by:

- Process cleanup
- Better reporting
- Training
- Policy clarification
- Rules-based automation
- Existing tool configuration
- Vendor capability
- Human staffing or role clarity
- Stopping low-value work

Good AI fit usually has:

- Repeated work
- Clear inputs and outputs
- Enough examples or source material
- Meaningful human review points
- Measurable value
- Tolerable error bounds
- A clear place where the output goes next

Weak AI fit usually has:

- Vague problem statement
- No owner
- No baseline
- High consequence of error with weak controls
- Poor data access
- One-off novelty
- No downstream user
- "AI because AI" rationale
