# Intelligence Stack Decomposition

For each AI idea, map the stack:

| Layer | Question |
| --- | --- |
| Purpose | What mission, business outcome, decision, or workflow improvement does this serve? |
| Sensing | What signals, inputs, events, files, systems, user actions, or documents must it observe? |
| Interpretation | What meaning, classification, extraction, synthesis, or diagnosis must it perform? |
| Decision | What recommendation, judgment, ranking, next step, or choice does it prepare? |
| Orchestration | What tools, systems, people, handoffs, or downstream workflows does it coordinate? |
| Learning | How does it improve from feedback, outcomes, exceptions, failures, and human review? |
| Govern And Assure | What evals, logs, permissions, rollback, review queues, and controls keep it safe? |

If one idea contains multiple distinct workflows, split it into smaller opportunities before value triage.
