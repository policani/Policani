Purpose

Resume Evidence Engine helps a user create job-search materials that are specific, truthful, and reusable. The project starts with a factual evidence base, then uses that evidence to score job descriptions and create targeted resumes, cover letters, and proof stories.

The package is intentionally blank except for templates and fictional samples. It should not contain a real user's private job history, preferences, credentials, references, or proof narratives when published.

Core Workflow

  1. Configure the project with builders/config-builder.html.
  2. Build the factual resume and evidence base.
  3. Create role-family variants and proof narratives.
  4. Score a job description against the evidence.
  5. Create final Markdown resumes or cover letters.
  6. Render final Markdown to DOCX when a capable agent environment supports it.
Conversion is a rendering step. The source Markdown owns the content. The renderer owns Word layout, spacing, bullets, hyperlinks, and validation.

First-Time Setup

  1. Open builders/config-builder.html.
  2. Complete the form and export project-config.md.
  3. Open builders/resume-builder.html to create core-canonical-resume.md.
  4. Use builders/case-study-builder.html to create proof stories from real work examples.
  5. Review docs/sample-builder-outputs/ if you want completed examples first.
  6. Ask Codex or another capable agent to run /setup and complete the evidence system.
  7. Only mark setup complete after the final checklist is true.

DOCX Handoff

The engine writes final resumes and cover letters as Markdown first. When an agent has a supported execution environment, it can render the Markdown to Word with the included renderer.

If rendering is unavailable, the correct fallback is to provide the final Markdown and state clearly that DOCX rendering could not be completed in the current environment.

Keep reference.docx. It is the Word style template used by the renderer.

GitHub Hygiene

Before publishing, keep only source templates, builder files, renderer files, instructions, public documentation, and fictional samples.

  • Do not commit outputs/, caches, QA reports, ZIP files, or temporary files.
  • Do not commit real resumes, cover letters, credentials, proof narratives, references, or job-search preferences.
  • Keep reference.docx and the intentional fictional sample DOCX.
