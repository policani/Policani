# Resume Writing Reference

## Purpose

Write resumes as decision artifacts, not career archives. Curate evidence for the target operating problem.

## Formatting rules

- ATS-friendly Markdown
- No tables, columns, text boxes, pipes, decorative formatting, icons, or graphics
- Plain section headings
- Straight quotes
- Hyphens for separators and ranges
- Sentence case
- Clickable visible text for links
- No raw URLs except where the visible URL is intentionally the link text

## Header

Use the configured candidate contact details. Keep it compact and readable.

## Role headers

Preferred format:

Insert Functional Title  
Insert Employer, Insert Location · Insert Month Year to Insert Month Year

Functional titles may clarify generic titles only when the work evidence supports them. Do not inflate level or authority.

## Bullet standard

Each bullet should:
- Start with a 2-4 word achievement anchor
- Use ` - ` after the anchor
- Be one sentence when possible
- Include scope or stakes where real
- Include an outcome, decision effect, control effect, or business consequence
- Stay under the correct employer and date range
- Avoid task inventory language

## Evidence rules

Do not invent:
- Metrics
- Titles
- Direct reports
- Budget authority
- P&L ownership
- Tools
- Customer ownership
- Certifications
- Dates
- Employers
- Outcomes

If a claim is adjacent but not direct, frame it as adjacent.

## Customization process

1. Score the role.
2. Select one variant by operating problem.
3. Identify 3-5 job-description operating concerns.
4. Map each concern to supported evidence.
5. Rephrase only where the evidence supports the concern.
6. Run validation gates.

## Validation gates

### Evidence integrity
Pass only if every claim is supported and correctly placed.

### Voice quality
Pass only if the resume reads as credible, specific, and human. Remove vague polish, hype, buzzwords, and symmetrical AI phrasing.

### Semantic coverage
Pass only if the resume addresses the role's real operating concerns without keyword stuffing.

### Hiring committee read
Ask whether a recruiter, hiring manager, and executive reviewer would understand the level, evidence, and risk in one scan.

## Credentials

Credentials support the case. They do not replace work proof. Include them only when the role values them or they improve credibility.
