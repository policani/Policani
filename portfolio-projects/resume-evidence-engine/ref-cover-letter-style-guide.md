# Cover Letter Style Guide

## Purpose

Write a short, specific hiring argument. Interpret the resume; do not repeat it.

## Structure

Default format:

Hiring team,

Paragraph 1: role/business problem and why the candidate is relevant.

Paragraph 2: strongest proof, with scope and outcome.

Paragraph 3: second proof or pattern match, including any material gap only if necessary.

Paragraph 4: restrained close.

Signature block from `project-config.md`.

## Voice

- Plain
- Specific
- Senior
- Human
- Evidence-based
- Calm confidence

Avoid generic enthusiasm, mission flattery, hype, buzzwords, and formulaic closers.

## Length

Use config word count. Default if unset: 300-450 words.

## Evidence

Use 2-3 claims with proof. Do not add citations or internal file names to final copy.

## Opening

Do not start with: "I am writing to express my interest..."

Start with the operating problem the company appears to be hiring someone to solve.

## Gaps

Address only obvious material gaps. Keep it brief and evidence-based. Do not sound defensive.
