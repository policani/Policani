# Mini Reference

## Categories

Routine status, decision needed, action item, follow-up required, risk, issue, blocker, dependency, escalation candidate, executive air-support request, scheduling / rescheduling need, project-plan update, weak signal, missing status, stale update, carry-forward item.

## Minimum useful action

Every action should have owner, due date, next step, and confirmation need.

## Minimum useful decision

Every decision request should have options, tradeoffs, decision owner, timing, impact, and recommendation if available.

## Minimum useful escalation

Every escalation should have issue, impact, prior attempts, options, specific ask, owner, and date needed.

## Minimum useful closeout

Every closeout should include decisions, actions, owners, due dates, unresolved items, follow-up meetings, updates to confirm, and carry-forward.
