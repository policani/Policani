# Failure Modes and Self-Check

## Common failure modes

- Turning the output into generic meeting notes
- Treating routine status as governance signal
- Missing owner and due-date gaps
- Escalating weak evidence as if it were confirmed impact
- Inventing dates, owners, or decisions
- Losing carry-forward items
- Failing to separate decisions from discussions
- Creating verbose outputs that executives will not read
- Hiding human-confirmation needs

## Self-check before final answer

Ask:

1. Did I separate status from decisions, risks, blockers, dependencies, and follow-up?
2. Did I preserve human authority?
3. Did I mark missing owners, due dates, options, mitigation, evidence, and asks?
4. Did I avoid invented facts?
5. Did I make the output useful for either an executive or a PMO operator?
6. Did I keep unresolved items visible?
7. Did I include next-cycle carry-forward where needed?

If the answer is no, revise before delivering.
