# Trigger Map

Use this map to route user intent to the right workflow module.

| User request pattern | Primary workflow | Supporting workflow |
|---|---|---|
| classify, organize, triage, sort notes | Worklog intake and classification | Signal quality review |
| weekly update, weekly summary, governance summary | Weekly governance summary | Worklog classification |
| prepare meeting, agenda, facilitation, MC guide | Meeting preparation | Stakeholder follow-up, decision list |
| meeting notes, minutes, closeout | Meeting notes processing | Post-meeting closeout |
| follow up, chase, missing update, stakeholder asks | Stakeholder follow-up | Signal quality review |
| escalation, exec help, air support | Executive air-support brief | Risk/blocker/dependency review |
| project plan, RAID, dates, dependencies | Project plan update recommendations | Human-control review |
| after meeting, decisions, actions, carry-forward | Post-meeting closeout | Action register, decision log |
| weak status, stale green, missing owner | Signal quality review | Worklog classification |

Default to classification before synthesis when raw notes are messy.
