# Stakeholder Follow-Up

## Purpose

Help the PMO operator chase updates, clarify ownership, confirm due dates, and coordinate next steps.

## Outputs

- Follow-up list by stakeholder
- Missing update requests
- Overdue action reminders
- Clarification requests
- Meeting scheduling / rescheduling list
- Draft email or chat text
- Priority order for follow-up work

## Grouping rule

Group by stakeholder or role. For each stakeholder, show:

| Field | Meaning |
|---|---|
| Stakeholder | Person or role |
| Initiative | Relevant initiative |
| Need | What is needed |
| Reason | Why it matters |
| Due date | Existing or proposed date |
| Suggested message | Draft follow-up text |
| Human confirmation | Any assumption to confirm before sending |

## Message drafting rules

- Be direct and neutral.
- Ask for the missing input, not a broad status update.
- Include the decision, action, risk, blocker, or dependency context.
- Do not blame the stakeholder.
- Do not send messages autonomously.
