# Project Plan Update Recommendations

## Purpose

Identify plan changes implied by the worklog and meeting notes.

## Outputs

- Suggested task updates
- Milestone changes
- Date changes
- Dependency updates
- Owner changes requiring confirmation
- RAID updates
- Decision-log updates
- Action-register updates
- Items requiring human confirmation before updating a plan

## Recommendation table

| Initiative | Current signal | Recommended update | Affected artifact | Confidence | Human confirmation needed |
|---|---|---|---|---|---|

## Rules

- Do not modify a live plan.
- Do not state an owner has changed unless provided by the user.
- Do not state a date changed unless provided; otherwise say `date change appears likely`.
- Separate observed facts from recommended updates.
- Identify which artifact should be updated: project plan, RAID, decision log, action register, calendar, stakeholder roster, or governance notes.
