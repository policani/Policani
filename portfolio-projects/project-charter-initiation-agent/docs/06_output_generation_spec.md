# Output Generation Specification

The system should produce three polished outputs.

## Markdown

Use Markdown for version control, pull requests, and collaborative edits. Keep tables simple and readable.

## HTML

Use a styled standalone HTML file for browser review, intranet posting, or PDF export through a browser. Include semantic headings, tables, and print-friendly CSS.

## DOCX

Use DOCX for formal sponsor review and signoff. Include:

- Title block
- Version and approval metadata
- Section headings
- Tables for roles, deliverables, risks, dependencies, success measures, governance, and milestones
- Signature/approval block

## Naming convention

- `project_charter.md`
- `project_charter.html`
- `project_charter.docx`
- `charter_quality_review.md`
