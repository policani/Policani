Resume Markdown Builder

Build a clean resume Markdown file for the Resume Evidence Engine. Everything runs in this page; no account, server, Git, Python, or install step.

Use this builder to create resume source files for the Resume Evidence Engine.

The engine requires core-canonical-resume.md at minimum. This is the factual master resume: work history, dates, titles, scope, skills, education, credentials, and evidence-backed achievements.

The engine performs better when you also create variant resumes, such as variant-resume-template-1.md, variant-resume-template-2.md, or role-family files like variant-resume-facilities-maintenance.md. Variants help the system match different job lanes without changing the truth source.

You can complete this builder yourself and download the Markdown file, or give your current resume and this builder page to Codex, ChatGPT, Claude, or a trusted teammate and ask them to generate the file using this builder as the template. Review the finished file for accuracy before using it.

Want a completed example first? Open sample core resume, sample variant resume, or a paired sample final resume Markdown and rendered Word file.

1. Resume Type

Choose the resume source file to create. The system always starts with Markdown, then a capable agent can render that Markdown to DOCX.

Core is the factual master resume. Variant is a reusable role-family resume. Final targeted resume is customized for one company and role.

For variants, use a role family such as "facilities maintenance" or "customer service". For final resumes, use "company - role".

Automatically follows the project naming convention so the system can discover the file later.

2. Header

These values become the resume header. Use one clear line per field and include only contact details that should appear in the resume.

Used as the main resume heading and in generated filenames.

A short role-positioning line, not a slogan. Example: Maintenance Technician.

City and state/region are usually enough. Example: City, State.

Use the address that should appear on employer-facing resumes.

Optional. Leave blank if the resume should not include a phone number.

Optional. Use the full profile URL so the Markdown link renders correctly.

Optional. Use only if there is a public portfolio or proof page you want employers to see.

3. Qualifications

Write the evidence-backed scan summary. Enter one proof-oriented point per line; each line becomes one bullet unless paragraph mode is selected.

Bullets are the default for ATS-friendly resumes. Paragraphs are useful for shorter executive summaries.

Use the heading you want in the resume. Common values: Qualifications, Profile, Summary.

Optional. Use a comma-separated list of skills, tools, services, or work capabilities.

4. Experience

Each role needs a title, employer, dates, and at least one evidence-based bullet. Add one role at a time and keep claims factual.

5. Education, Credentials, Skills

One line becomes one bullet. Keep education, credentials, and skills truthful, current, and role-relevant.

Enter one degree, diploma, program, apprenticeship, or relevant coursework item per line. Example: High school diploma or GED - Example School.

Enter one certification, license, training program, or credential per line. Include only items you can verify.

Use concise skill clusters. Separate multiple entries with commas, or put one skill group per line.

Optional. Add languages, publications, awards, volunteer work, or other relevant notes, one per line.
