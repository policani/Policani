Resume Evidence Engine - Config Builder

Use this file locally in your browser. It does not send data anywhere. Complete the form, review the Markdown preview, then export project-config.md. The project workflow is fixed: create Markdown first, then render that Markdown to DOCX when a capable agent can do the conversion.

How settings work

Most fields become simple Value: lines in project-config.md. If a required field is blank, the generated config will contain Insert... so the agent can stop and tell the user exactly what to finish.

For multi-entry text boxes, use comma-delimited lists. Line breaks and semicolons also work; the exported config normalizes them to commas. Dropdown values marked (default) are the recommended starter settings.

2. Job-search target

Target roles are the jobs the candidate wants. Target audiences are the people or organizations most likely to hire them. Clear comma-delimited lists help the agent avoid assuming a corporate or technology audience.

Use a comma-delimited list of roles the candidate would actually apply for.

Examples: maintenance technician, facilities maintenance, apartment maintenance, equipment repair, building engineer

Use a comma-delimited list of hiring audiences, workplaces, customers, or employer types.

Examples: property managers, facilities supervisors, plant managers, school districts, homeowners, hiring managers
