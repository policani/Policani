# Evidence Map Template

Purpose: Route job-description signals to proof. This file prevents unsupported claims and helps select the right resume variant, proof narrative, and guardrail.

## Maintenance rule

Add entries only when source evidence exists in:
- `core-canonical-resume.md`
- a completed proof narrative or work example
- `ref-credentials.md`
- another approved evidence file

Do not add wish-list capabilities.

## Entry format

### Insert JD signal or capability

- Source: Insert case file, canonical anchor, credential file, or variant
- Supported claims: Insert claims allowed from source evidence
- Resume variant: Insert variant filename
- Best proof narrative: Insert proof narrative filename or None
- Guardrails: Insert what not to claim
- Recency note: Insert whether evidence is recent, historical, or mixed
