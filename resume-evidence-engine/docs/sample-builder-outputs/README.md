# Sample Builder Outputs

These files are fictional examples of what the browser builders can produce.
Use them as models for tone, structure, and level of detail.

The examples are intentionally plain. A good job-search file does not need to
sound fancy. It needs to be clear, truthful, and easy for another person to
understand.

## Files

- `sample-core-canonical-resume.md` - example master resume file.
- `sample-variant-resume-customer-operations.md` - example role-family resume variant.
- `sample-final-resume-customer-operations-coordinator.md` - example final targeted resume Markdown.
- `sample-final-resume-customer-operations-coordinator.docx` - Word version rendered from the matching Markdown source.
- `sample-case-study-service-request-backlog-reset.md` - example process-fix case study.
- `sample-case-study-refund-error-reduction.md` - example money, risk, and customer-impact case study.

## How to Use These

Open the matching builder in `builders/`, read the sample output beside it, and
replace the fictional details with your real facts.

To see the Markdown-to-DOCX workflow, compare
`sample-final-resume-customer-operations-coordinator.md` with
`sample-final-resume-customer-operations-coordinator.docx`.

That Word file was rendered from the Markdown source with the bundled project
renderer. The content stayed in Markdown; the renderer handled the Word layout,
spacing, bullets, and hyperlink.

Do not copy numbers, employers, titles, dates, certifications, or outcomes from
these samples unless they are true for you.
