# DOCX Generation Notes

The generated DOCX example in this folder is produced from the synthetic scored portfolio output using `python-docx`.

A production implementation should:

- Use Word heading styles for navigation.
- Keep score tables concise enough for portrait layout or use a landscape section for wide matrices.
- Include criteria and weights in an appendix.
- Exclude sensitive source details from public documents.
- Visually inspect rendered pages before distribution.

The DOCX is an example report, not an official portfolio decision record.
